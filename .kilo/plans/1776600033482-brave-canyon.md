# Plan: Enable Comprehensive Action Debug Logging

## Problem
Debug logs are fragmented across different hooks and not all actions are logged:
- `usePackages` logs are passed to DebugPanel but only fetch operations
- `usePackageInstaller` callback in Browse.tsx just uses `console.log()` - doesn't reach DebugPanel
- `useInstalledList` has no debug logging
- Module loading events aren't logged

## Solution
Create a centralized debug logging system so all actions are logged in one place.

## Implementation Steps

### Step 1: Create Debug Logger Module
Create a simple module-level logger:
- File: `src/debugLogger.ts`
- Store logs in module-level array
- Export `addLog(msg)` function
- Export `getLogs()` to read logs
- Export `clearLogs()` function

### Step 2: Update App.tsx
- Import `getLogs()` and use in state for DebugPanel
- Refresh logs periodically or after actions

### Step 3: Update Hooks to Use Centralized Logging

**usePackage.ts:**
- Import `addLog` from context
- Replace local `setDebugLogs` with `addLog`

**useInstalledList.ts:**
- Add debug logging using centralized system
- Log: start of fetch, success with count, errors

**usePackageInstaller.ts:**
- Already has log callback parameter - modify caller in Browse.tsx to pass centralized `addLog`

### Step 4: Add Module Loading Logs
Add logging in `modules-loader.ts` for:
- Module load attempts
- Load failures
- Success indicators

### Step 5: Verify
- Only `debugLogs` prop passed to DebugPanel
- All actions logged: browsing, installing, listing installed, module loading
